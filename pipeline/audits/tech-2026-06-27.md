## 2. Architectures i18n observées

Trois patterns coexistent dans le réseau, **tous fonctionnellement conformes** (aucun n'est le bug « fausses catégories ») :

1. **Arbre `app/en/` séparé** (le pattern doctrinal de référence) : suv, familiale, utilitaire, 7-places, quel-operateur, fournisseur-energie, assurances-auto, neobanque.
2. **`app/[locale]/` + middleware** (variante propre, plus DRY) : voiture-electrique, carte-credit.
3. **Middleware rewrite `/en/*` → routes FR + header locale** (sans arbre `app/en/`) : meilleure-voiture.be.

Tous portent : lecteurs miroir EN, mapping FR↔EN, `LangSwitch` monté page↔page, hreflang réciproque + x-default, garde anti-fausses-catégories (`RESERVED_LOCALE_DIRS` / `isFrCategory`).

## 5. Top 5 des urgences

1. **meilleur-fournisseur-energie.be — hreflang réciproque + x-default sur les articles.** Ajouter `alternates.languages` (fr + en + x-default) dans `generateMetadata` de `app/(site)/[article]/page.tsx` pour les 3 slugs traduits. Aujourd'hui le lien hreflang est unilatéral (EN→FR seulement) : Google reçoit un signal contradictoire. *(+ combler la parité EN : 3/26.)*

2. **meilleure-voiture-utilitaire.be — articles FR absents du sitemap.** `app/sitemap.ts` n'émet aucune des 8 URLs racine `/[slug]` côté FR — le contenu principal du site n'est pas déclaré aux moteurs. Itérer sur `getAllStandaloneArticles()` (FR) comme c'est déjà fait pour l'EN.

3. **quel-operateur-choisir.be (RÉFÉRENCE) — fichier EN égaré → hreflang vers 404.** Déplacer `content/blog/en/best-mvno-alternative-operators-belgium.mdx` vers `content/blog/en/mobile/`. Rétablit la parité (9↔9) et supprime un hreflang FR→EN pointant une 404. Critique car ce site sert de modèle au réseau.

4. **meilleure-voiture-familiale.be — violation « JAMAIS de NL ».** Supprimer `content/blog/nl/beste-gezinswagen-2026.mdx` (+ dossier `content/blog/nl/`), le placeholder 0 octet `content/blog/en/monospaces/best-family-mpv-2026.mdx` et le composant mort `components/layout/LangSwitch.tsx`.

5. **meilleure-voiture-7-places.be — mapping i18n quasi-vide.** Compléter `lib/i18n/article-slugs.ts` (1 entrée seulement) en ajoutant au minimum `kia-ev9-vs-peugeot-e-5008` (paire FR+EN déjà sur disque, actuellement sans hreflang réciproque ni LangSwitch). *(Voir aussi le bug `lang="fr"` figé sur les pages `/en` du même cluster utilitaire.)*
