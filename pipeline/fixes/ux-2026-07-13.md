## Corrigé

### ❌ Images — Covers de catégorie générées **et câblées** (correctif fantôme levé)
- **Cause racine** : les 5 covers de `public/images/categories/*.webp` existaient mais
  **aucun composant ne les consommait** (le `hub-hero` de `CategoryView` ne rendait que
  du texte). En prime, les anciennes étaient en **4:3 hors-DA** et `simulateur.webp`
  affichait un **écran à dégradé arc-en-ciel néon** — violation de `signature.forbidden`.
- **Fix** :
  - Générées **5 covers 16:9** dans la DA éditoriale papier/cardinal (sobres, sans
    voiture accidentée, sans logo d'assureur) : `simulateur`, `comparatifs`, `garanties`,
    `prix`, `profils` → `public/images/categories/{slug}.jpeg`.
  - `lib/blog.ts` : ajout `getCategoryCover(slug)` (résout `.jpeg|jpg|webp|png` avec
    `existsSync` — jamais de chemin cassé, `.jpeg` prioritaire sur le `.webp` legacy) et
    `getCategoryCoverAlt(slug)` (alt = `niche.categories[].description`).
  - `CategoryView` : `hub-hero` passé en **2 colonnes** (texte + cover `next/image`
    `priority`), empilé < 860px, cover masquée < 520px. **Rendu vérifié dans le chemin**
    (le composant consomme réellement la cover).
  - `volteo-hub.css` : styles `.hub-hero--cover / -inner / -text / -cover`.
- **Câblage** : ✅ prouvé (chemin de rendu `CategoryView` → `getCategoryCover`).
- Commits : `d3c2d09` (code), `550f711` (css), `33fee3c` (assets canoniques).

### ❌ Images — ~2 images in-body par article (FR + EN)
- Les 2 articles (`comparatifs/simulateur-assurance-auto-belgique.mdx` + miroir EN)
  avaient une featured unique mais **0 image in-body**.
- **Fix** : insertion de **2 `<ArticleImage>`** (composant déjà câblé dans le pipeline
  MDX de `ArticleView`), **réutilisées du pool** de covers (`prix.jpeg` dans la section
  facteurs de prix, `garanties.jpeg` dans la section RC/omnium), **alt descriptifs** +
  légendes, `next/image`. **Texte de l'article inchangé** (invariant anti-perte respecté).
- FR + EN. Commits : `a0a767a` (FR), `a915910` (EN).

### ❌ DA / tokens — Purge des hex en dur de `volteo.css :root` (+ `volteo-fil.css`)
- `--elec/--gaz/--green/--water/--mobi` (jaune/orange/**vert**/bleu/violet du skin
  « Électrique ») **rendaient** hors-palette : bandeau newsletter `.news` en **VERT
  #1AA35F**, point « live » vert, et `#E0524D` en dur dans `volteo-fil.css`.
- **Fix** : aliasés sur `--accent-1..5` (pilotés par `niche.config.palette` → globals) ;
  variantes `-soft` et couleurs de texte des tags dérivées en `color-mix` (contraste AA
  conservé) ; `#E0524D` → `var(--primary)`. **Plus aucun hex de DA** dans ces fichiers.
- ⚠️ **Footprint réseau** : ces tokens étaient identiques sur **tous les sites du
  template** (même bandeau vert). → **à remonter à `emd-template`** (cf. Reporté).
- Commit : `3a0b188`.

### ⚠️ A11y — `alt` descriptifs (featureImageAlt) partout
- `featureImageAlt` (rédigé dans le frontmatter) était **ignoré** : les covers passaient
  `alt={title}` (décrit le lien, pas l'image).
- **Fix** : `featureImageAlt` ajouté à `ArticleMeta` + `parseMeta` (additif, optionnel).
  Les **6 consommateurs de cover** énumérés et alignés sur `alt = featureImageAlt ?? title` :
  `CategoryView.Cover`, `FilHome.Cover`, `ArticleCard` (featured + normale),
  `MagazineHome.Cover` (2 branches), `MarcheHome.Cover`. `ArticleView` (hero décoratif
  `aria-hidden`, `alt=""`) laissé tel quel volontairement.
- Commits : `d3c2d09`, `550f711`, `0e5f508`.

### ⚠️ Responsive — `body { overflow-x: hidden }` neutralisé
- La règle **masquait** les débordements au lieu de les corriger et rendait « zéro scroll
  horizontal » invérifiable.
- **Fix** : `body { overflow-x: visible }` (dans `volteo-hub.css`, chargé après globals) ;
  seul `html { overflow-x: clip }` (ne casse pas `position:sticky`) reste comme garde.
  Débordements réels traités à la source : input newsletter `60vw → 100% + min-width:0`,
  `overflow-wrap: anywhere` sur titres de cartes/rubriques/hub-hero.
- Commit : `3a0b188` (+ `550f711`).

### ⚠️ A11y — cible tactile du burger ≥ 44px
- `.nav-burger` padding `8px` + icône 26px ≈ 42px → **padding `10px`** ≈ 46px (WCAG 2.5.8).
- Commit : `3a0b188`.

### ⚠️ A11y — `lang` du document sur `/en`
- `<html lang="fr">` en dur ; `/en/*` restait déclaré `fr` au niveau **document** (le
  contenu, lui, était déjà couvert par `<div lang="en">`).
- **Fix (le moins invasif)** : composant client `HtmlLang` monté dans `app/en/layout.tsx`
  → pose `document.documentElement.lang = 'en'` (restauré à `fr` au démontage). Le
  `<div lang="en">` est conservé (couverture no-JS pour l'AT).
- **Choix assumé** : lire `headers()` dans le root layout aurait **déopté tout le SSG du
  blog** en rendu dynamique → tradeoff refusé pour un attribut de langue. Le vrai
  correctif statique (arbre `app/[locale]` + `<html lang>` par segment) est un **gros
  chantier de routing** → **reporté** (risque build élevé en run non supervisé).
- Commit : `1223df0`.
