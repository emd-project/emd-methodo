## Détail vérifié + correctifs prescrits

### meilleure-voiture.be — sitemap cassé (BLOQUANT) + slug parasite

**Câblage vérifié :** `app/` ne contient que `app/(site)` → **aucun `app/en`** ; toutes les routes `/en` sont des 404. `niche.config.ts` : `defaultLocale: 'fr'`, `locales: ['fr','en']`, `localePrefix: 'as-needed'`, `localePath('en', p)` → `/en${p}`. Donc le sitemap actuel **émet bien des `/en/...` morts**. Il n'inclut que `getAllStandaloneArticles(loc)` (`@/lib/articles`) → **les articles de blog (`content/blog/[cat]/*.mdx`, lus par `getAllArticles()` de `lib/blog.ts`) sont absents**. Les deux constats de l'audit sont confirmés sur la source rendue.

**Fix prescrit — `app/sitemap.ts`** (mono-locale FR + ajout blog ; `getAllArticles` et signature locale vérifiés dans `lib/blog.ts`) :

```ts
import type { MetadataRoute } from 'next'
import { niche, localePath } from '@/niche.config'
import { getAllStandaloneArticles } from '@/lib/articles'
import { getAllArticles } from '@/lib/blog'

const SITE_URL =
  process.env.NEXT_PUBLIC_SITE_URL ?? `https://${niche.domain}`

const STATIC_SECTIONS: { path: string; changeFrequency: MetadataRoute.Sitemap[number]['changeFrequency']; priority: number }[] = [
  { path: '/', changeFrequency: 'weekly', priority: 1 },
  { path: '/blog', changeFrequency: 'daily', priority: 0.9 },
  { path: '/comparer', changeFrequency: 'weekly', priority: 0.8 },
  { path: '/quiz', changeFrequency: 'monthly', priority: 0.8 },
  { path: '/deals', changeFrequency: 'daily', priority: 0.7 },
]

export default function sitemap(): MetadataRoute.Sitemap {
  const entries: MetadataRoute.Sitemap = []
  const loc = niche.defaultLocale // site mono-locale : aucun arbre app/en

  for (const { path, changeFrequency, priority } of STATIC_SECTIONS) {
    entries.push({
      url: `${SITE_URL}${localePath(loc, path)}`,
      lastModified: new Date(),
      changeFrequency,
      priority,
    })
  }

  for (const a of getAllStandaloneArticles(loc)) {
    entries.push({
      url: `${SITE_URL}${localePath(loc, `/${a.slug}`)}`,
      lastModified: new Date(a.updatedAt ?? a.publishedAt),
      changeFrequency: 'monthly',
      priority: 0.7,
    })
  }

  // Articles de blog (étaient absents du sitemap)
  for (const a of getAllArticles(loc).filter((x) => !x.standalone)) {
    entries.push({
      url: `${SITE_URL}${localePath(loc, `/blog/${a.categorie}/${a.slug}`)}`,
      lastModified: new Date(a.updatedAt ?? a.publishedAt),
      changeFrequency: 'monthly',
      priority: 0.7,
    })
  }

  return entries
}
```

> Note : `/simulateur` a été retiré des sections statiques (`niche.config` → `simulator.enabled: false`, la route n'existe pas). À confirmer côté routing avant build.
> Alternative si l'EN est réellement provisionné un jour : reconditionner l'émission `/en` à l'existence effective des routes (pattern `meilleur-suv.be`).

**Fix prescrit — `lib/i18n/article-slugs.ts`** (anti-footprint : l'entrée `'meilleur-suv-belgique-2026' → 'best-suv-belgium-2026'` est copiée d'un autre site et n'appartient pas à voiture.be) :

```ts

export const articleSlugFrToEn: Record<string, string> = {}
```

(Les fonctions `articleSlugInOrNull` / `translateArticleSlug` gèrent déjà l'objet vide via `?? null` — aucun autre changement requis.)

**Décision i18n :** porter un `app/en` complet (pattern `quel-operateur-choisir.be`) = **build-risquant** → **i18n à porter (build check requis)**, à faire en incréments compilables (légal EN → blog EN → home EN). À défaut, requalifier le statut `sites.csv` en `fr` (mono-locale) et appliquer le seul fix sitemap ci-dessus.

Mais `lib/blog.ts` est la **version mono-locale** : `BLOG_DIR`/`ARTICLES_DIR` constants figés, **aucun lecteur miroir `…En()`**, aucune signature `locale`, aucun `RESERVED_LOCALE_DIRS`. `content/blog/` ne contient que des catégories FR (`comparatifs`, `guides`, `jeunes-conducteurs`, `simulateurs`, `types-assurance`) → **`content/blog/en` absent**.
