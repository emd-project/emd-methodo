### ⛔ Reporté — action INFRA, hors code (priorité n°1 de l'audit)

**1. Le domaine `simulateur-assurance-auto.be` ne répond pas** (`ERR_CONNECTION_REFUSED` sur `www.`).
Toutes les URLs absolues (canonical, hreflang, `og:url`, `og:image`, sitemap) sont émises sur ce host
via le fallback `https://www.${niche.domain}` — `NEXT_PUBLIC_SITE_URL` n'est pas défini.

**Non corrigé volontairement.** Hardcoder `https://simulateur-assurance-auto-be.vercel.app` dans le code
figerait les canonicals du site sur l'alias Vercel et devrait être défait au branchement du domaine
(anti-idempotent, et un canonical vers un alias de déploiement est pire que le statu quo).
**Action humaine requise, l'une des deux :**
- brancher le domaine sur le projet Vercel (DNS) — solution durable ; **ou**
- définir la variable d'env Vercel `NEXT_PUBLIC_SITE_URL=https://simulateur-assurance-auto-be.vercel.app`
  en attendant (aucun code à changer : tout le repo lit déjà cette variable en priorité).

À vérifier aussi au branchement : le code émet `https://www.` + domaine — s'assurer que **`www`** est bien
le host canonique servi (sinon aligner `niche.domain` / `NEXT_PUBLIC_SITE_URL` sur l'apex).
